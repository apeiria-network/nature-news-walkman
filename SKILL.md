# Nature News Digest

Support 3 independent command modes for Nature news reading and listening practice, instead of one fixed bilingual output format.

## When to Use

- User asks for "Nature最新新闻" / "Nature news"
- User wants English summaries of Nature news
- User wants full English text of Nature news
- User wants English listening practice with Nature news audio
- User uses `/nature-news-brief`, `/nature-news-text`, or `/nature-news-sound`

## Command Modes

### `/nature-news-brief [user requirements]`
- Output only an English summary for each news item
- Each summary should be about **120-180 words**
- The summary is generated by the LLM from the fetched English news text

### `/nature-news-text [user requirements]`
- Output the **full original English text** for each news item
- Do not replace the news text with a summary unless the user explicitly asks

### `/nature-news-sound [user requirements]`
- Output the **full original English text** for each news item
- Generate and send **English full-text audio** for each news item
- Audio save-path logic stays the same as the current skill

## User Requirements After the Command

Any text after the command is part of the user's request and should be preserved.

Examples:
- `/nature-news-brief easier English, climate topic, scientific tone`
- `/nature-news-text cancer biology, keep terminology, also add Chinese key points`
- `/nature-news-sound choose space-related news, suitable for listening practice`

The command name controls the output mode. The trailing text controls customization.

## Shared Retrieval Logic

The skill now has 3 parallel command modes, so do **not** treat the task as a rigid step-by-step user workflow. Instead:
- First determine which command mode the user invoked: `brief`, `text`, or `sound`
- Then apply the corresponding output rules for that mode
- Reuse the same Nature news search/fetch logic across all 3 modes
- The **agent** decides whether to reuse the latest fetched news batch or do a fresh search
- Only generate audio for `sound` mode

## Workflow

### Shared News Retrieval

Across all 3 command modes:

1. Use **WebSearch** and/or **WebFetch** to identify the latest top Nature news from `https://www.nature.com/news`
2. Select the top 3 most relevant **news items** based on recency, editorial prominence, and any user requirements
3. For each selected news item, record: title, URL, author (if available), date, and brief description
4. Fetch the **full original English text** of each selected news page
5. If a fetch fails, retry once. If it still fails, skip that news item and note the failure

When lightweight cache reuse is available:
- The **agent** decides whether to reuse the latest fetched **news batch** instead of searching again
- Reuse is especially preferred when the user switches between `brief`, `text`, and `sound`
- If the user clearly asks to refresh, re-search, switch topic, or use a different batch, the agent should do a fresh retrieval instead
- Preserve the user's original trailing text requirements so later commands can reuse them when appropriate
- The code helper only reads and writes the cache file; it does **not** decide reuse policy

### Output Rules by Command Mode

#### `brief`
For each selected news item:
- Output `News {N}: {English Title}`
- Include source URL and date if available
- Output one English summary of about **120-180 words**
- The summary must be generated from the fetched English text
- Do not output the full English text unless the user explicitly asks
- If the user asks for Chinese support, add Chinese notes or key points as an extra section

#### `text`
For each selected news item:
- Output `News {N}: {English Title}`
- Include source URL and date if available
- Output the **full original English text**
- Do not replace the full text with a summary unless the user explicitly asks
- If the user asks for Chinese support, add it after the English original text

#### `sound`
For each selected news item:
- Output `News {N}: {English Title}`
- Include source URL and date if available
- Output the **full original English text**
- Generate English full-text audio for that news item
- Send the generated `.mp3` file to the user
- If the user asks for Chinese support, add it after the English original text

### Audio Generation Rules (`sound` only)

For each selected news item in `sound` mode:

1. Prepare the English text for TTS:
   - Remove markdown formatting (*, **, #, etc.)
   - Remove credit lines ("Credit: ...")
   - Remove image references
   - Collapse excessive whitespace

2. Before generating audio, prepare the Python runtime in the skill root:
   - Always prefer an existing `.venv` under the skill root
   - A reusable `.venv` must use Python **3.10–3.14**
   - If that `.venv` is missing required dependencies, install the missing dependencies into **that same `.venv`**
   - If no suitable local `.venv` exists, create `.venv` under the skill root with Python **3.10–3.14**
   - Install all runtime dependencies for this skill only into the skill-local `.venv`
   - Do **not** install dependencies into system Python, user site-packages, or any environment outside the skill root

3. Generate audio with **auto-fallback** (gTTS → edge-tts):

   **Primary: gTTS** (requires Google access):
   ```python
   from gtts import gTTS
   tts = gTTS(text=cleaned_english_text, lang='en', slow=False)
   tts.save(f'D:/nature-news-digest-sounds/Nature_Article{N}_English.mp3')
   ```

   **Fallback: edge-tts** (works in China, no VPN needed):
   ```python
   import edge_tts, asyncio
   async def _gen(text, path):
       com = edge_tts.Communicate(text, 'en-US-AvaNeural')
       await com.save(path)
   asyncio.run(_gen(cleaned_english_text, f'D:/nature-news-digest-sounds/Nature_Article{N}_English.mp3'))
   ```

   Or use the unified helper (auto-fallback built in, local `.venv` enforced):
   ```python
   from scripts.nature_digest import generate_tts_audio
   success, engine = generate_tts_audio(text, output_path, lang='en', engine='auto')
   ```

   **Fallback logic**: Try gTTS first. If gTTS fails (e.g., Google blocked in China), automatically switch to edge-tts. Log which engine was used.

4. Save each file as: `Nature_Article{N}_English.mp3` in `D:/nature-news-digest-sounds` by default; if `D:` is unavailable or not writable, fall back to `C:/nature-news-digest-sounds`
5. When presenting results, send newly generated audio as actual `.mp3` files. Do **not** send audio content as text, transcript, or a language-only description.
6. If file delivery fails, explicitly tell the user the current saved file path so they can retrieve the audio manually.

### Present Results

1. Send the generated text result directly in the conversation
2. For `sound`, send newly generated audio to the user as `.mp3` files
3. If file delivery fails, tell the user the current saved path of each generated audio file
4. Briefly mention whether the current news batch was freshly fetched or reused from cache
5. Note any skipped news items, paywall limitations, or fetch failures

## Error Handling

- **WebFetch fails**: Retry once. If still failing, skip the news item and inform the user.
- **Cache read fails**: Ignore the cache and do a fresh retrieval.
- **Cache write fails**: Continue the response, but later command reuse may not be available.
- **Local `.venv` missing or incomplete**: Reuse the existing skill-local `.venv` when valid; otherwise create `.venv` under the skill root with Python 3.10–3.14 and install any missing required dependencies only into that `.venv`.
- **Local `.venv` version invalid**: If the existing skill-local `.venv` is not Python 3.10–3.14, do not use any external environment; report the issue and recreate the local `.venv` under the skill root.
- **gTTS fails (Google blocked)**: Automatically fall back to edge-tts within the prepared skill-local `.venv`.
- **edge-tts also fails**: Inform the user that both TTS engines failed and suggest checking network or the skill-local `.venv` dependencies.
- **Audio file delivery fails**: Inform the user that file delivery failed and provide the current saved path for each generated audio file.
- **Paywalled content**: Extract all freely visible text. Add a note at the end of that news item.
- **Empty news text**: Skip it and fetch the next most relevant Nature news item.
